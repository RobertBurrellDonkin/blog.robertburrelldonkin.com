The declarative paradigm is based on describing what should be achieved as opposed to an imperative set of commands.

As we found when we created Maven, declarative tooling is a lot of effort to write.

But leaving the *how* to vary allows the tooling to evolve and improve continuously over time behind the scenes.

And declarative tools have become the rule rather than the exception.
